小白看完都会的红黑树真解

发表于2022-04-22|更新于2022-04-23
|阅读量:

简介

红黑树（Red Black Tree，RBT）是一棵二叉搜索树（BST），每个结点都拥有自己的颜色，红色（Red）或者黑色（Black）

通过一些条件进行约束，红黑树达到了近似平衡。

在java中的jdk1.8后的Hashmap与concurrentHashmap就用到了红黑树的性质

平衡二叉树 vs 二叉搜索树

在学习二叉搜索树时，我们了解到在极端条件下二叉搜索树会退化成一条链，这个时候查找的时间复杂度O(log2N)退化成O(N)，由于二叉搜索树的查找效率取决于树的高度，因此保持树的高度最小，即可保证树的查找效率。因此就有了AVL树。

平衡二叉树 vs 红黑树

红黑树使用非严格的平衡换取增删结点时的旋转次数降低，而 AVL 严格平衡，意味着在增加或删除结点时，根据情况的不同，旋转的次数比红黑树要多

AVL 的结构相较于红黑树更为平衡，因此 AVL 的查询效率更高

总体来说，红黑树的统计性能高于 AVL，但如果是频繁查询而少次增删的场景，使用 AVL 更加合适

红黑树基本概念

红黑树的结构

属性

红黑树的节点一般包含五个属性:

  • color: 结点的颜色
  • key: 结点的大小（这个大小并不是数值上的大小，而是比较的大小）
  • **left:**左孩子结点
  • right: 右孩子结点
  • p: 父结点

相比于 AVL 树，红黑树的属性只增加了color

区别

他俩还有一个不同，AVL 以真实存在的结点作为叶子结点，而红黑树以空指针 NIL 作为叶子节点

但实际上我们只需要考虑红黑树的内部节点即可(即带有key值的节点)，因此接下来的红黑树结构我们一般都将叶子节点忽略掉

性质

红黑树有以下五种性质：

  1. 每个结点只能是红色或者黑色的
  2. 根节点是黑色的
  3. 每一个叶子结点（NIL）都是黑色的
  4. 如果一个结点是红色的，那么它的两个子节点都是黑色的
  5. 对任意结点，从该路径到其所有后代叶结点的简单路径上，均包含相同数目的黑色结点

这五种性质非常之重要,请读者牢记！！！后面对于红黑树的所有操作都是基于这几条性质

黑高

黑高这个概念是由性质5而引申出来的，下面建议对着图进行理解

图中的数字表示从当前结点 x 到叶结点路径上经过的黑色结点数目（不包括当前结点），称为该结点的黑高（black-height），记为 bh(x)

根据性质5，黑高的值是明确的，因为从该结点出发，下降到叶结点的任意简单路径的黑色结点个数都相同

例如：

  • bh(A)=1 ，因为只经过了一个 NIL。

  • bh(C)=2，因为经过了 A 和 NIL / 或是 B 和 NIL

  • bh(D)=2，因为不管走左边还是右边，都经过两个黑色结点

需要注意的是，计算黑高时不能将结点自身计算进去，这在黑高的定义中也明确声明了

定理

定理:一棵有 n 个内部结点的红黑树的高度至多为 2lg(n+1)

在《算法导论》中有对该定理进行进行十分严格的证明，大佬可以直接移步至1.3.2严格证明，如果看不懂可以回来看我对该定理的理解

小白理解

我们要明确，我们对该定理的证明是为了去求得一棵 N 结点红黑树的时间复杂度，因此转换成了该定理的证明

目标：求得 N 结点红黑树的最大高度

首先，我们再回去看一眼红黑树的性质5，我们会发现，红黑树的高度是与黑高息息相关的，你想，平衡二叉树的高度与什么有关，是与它的左子树与右子树的深度之差，他的绝对值不能超过1，而红黑树的高度与什么有关，与性质5有关。

那么接下来该如何求黑高的范围呢，我们把目光投至性质4，我们发现，红色节点下已经要有两个黑色节点，让我们还回忆一下高中求极限值的做法

  • 特殊值
  • 极端情况

（咳咳，这是完全不会做的情况下，想到的办法)

因此，我们会想到，在极端情况下，即一个树是红黑交替的，它的高度就等于两倍的黑高，即H(x)的最大值为 2bh(x)，如下图

那么又可以把目标换一换，换成求 N 结点红黑树的最大黑高

N 个结点堆成一棵树，路子可多了去了，到底什么情况下才能拥有最大的黑高，好求吗？不好求。

但是把这句话反过来看就容易多了。

求N 结点红黑树的最大黑高，等价于，求黑高为 bh(x)的以x作为根节点的红黑树所包含的最少结点数

以 x 为根，黑高为 bh(x) 的子树所包含的最少结点数

在这里，我们将红黑树分为两种情况

  • 全黑（基于性质5，我们可以想到全黑的红黑树必定是完全二叉树，否则黑高就不相等了）
  • 包含红色节点

这里再次提醒读者，黑高bh(x)的计算是不包括节点本身的，但是会考虑NIL节点的黑色

接下来进行计算

1、全黑

上面指出了一棵全黑的红黑树必定是完全二叉树，所以黑高为 bh(x)的子树如果是全黑，那么它的结点数量是可以定量计算的。

一棵高度 h=bh(x)的完全二叉树的结点数量 N=2bh(x)−1(读者这里如果不理解，可以自行画下图)

2、包含红色节点

假设我们在第k(k>1,性质2)层出现了第一个红色节点

那么这时 bh(x)≥k−1 ，同时结点数量 N≥2^k−1^(当第 k 层只有一个内部结点且为红色时取等号)

而如果是全黑的情况，在 bh(x)=k−1的条件下，

结点数量 N=2bh(x)−1=2^k−1^−1 ，一定小于上面的 2k - 1

所以可以得出结论：一棵黑高为 bh(x) 的树的结点数量 N≥2^bh(x)^−1

可能这样说不太好理解，那我们来举一个例子:

  • 我们先画一个bh(x) = 3的全黑节点的红黑树

现在我们想做的是在黑高不变的前提下添加红色节点

那么有两种做法:

  • 在底层添加新的红色节点
  • 将面某个节点染红(不能是根节点，性质2)

若我们在底层添加红色节点，或增加节点数量，因此我们考虑用染色

​ 举个:chestnut:,我们现在将节点8进行染色

​ 这时候你会发现，右子树的黑高已经被破坏了，我们需要在节点6和节点9的下面各添加一个黑色节点才能保证黑高的平衡，但是这也会 增加节点数量

​ 综上所述:一棵黑高为bh(x)的树只有在结点全黑时拥有的结点数最少

​ 进而可以推断出:一棵黑高为 bh(x) 的树的结点数量 N≥2^bh(x)^−1

有了这个结论，我们就可以计算N 结点的红黑树的最大高度了

得出结论：N 结点的红黑树最大高度是 2lg(N+1)

所以红黑树的搜索时间复杂度是 O(lgn)

严格证明

以下内容出自（照搬） 《算法导论》第三版（机械工业出版社）

证明：一棵有 n 个内部结点的红黑树的高度至多为 2lg(n + 1)

先证明以任一结点 x 为根的子树中至少包含 2bh(x) - 1 个内部结点。要证明这点，需要对 x 的高度进行归纳。

  1. 当x的高度为0，则x必为叶结点（T.nil),以x为根结点的子树至少包含2bh(x)-1 = 20 - 1 = 0个内部结点。结论正确

  2. 假设x为红黑树的某一内部结点，且它高度h>0，那么它的黑高度就是bh(x)，但是它的两个孩子结点呢？这个就根据它们的颜色来判断了： 如果x有一个红色的孩子y，那么y的黑高度bh(y)=bh(x)，看看上面对黑高度的定义你就明白了——既然它是红色的，那么它的黑高度就应该和它父亲的黑高度是一样的；
    如果x有一个黑色的孩子z，那么z的黑高bh(z)=bh(x)-1，这个怎么解释呢，因为它自己就是个黑结点，那么在计算它的黑高度时，必须把它自己排除在外（还是根据定义），所以它是bh(x)-1。

    那么对于每一棵子树，其黑高至少为bh(x)-1，根据归纳法假设子树至少有2bh(x - 1)-1 个内部结点。所以，对x来说，它所包含的内部结点个数“至少”为两个孩子结点所包含的内部结点数，再加上它自己，于是就为(2bh(x - 1)-1 ) + (2bh(x - 1)-1 ) + 1 = 2bh(x )-1 ，结论正确。

综上所述……

为完成引理的证明，设 h 为树的高度。根据性质4，从根到叶结点（不包括根结点）的任何一条简单路径上都至少有一半的结点为黑色。因此，根的高度至少为 h / 2；于是有

​ n≥2^h/2^ -1

然后就和上面的公式计算一样，得出h≤2lg(n+1)

INSERT（插入）和 DELETE（删除）在含 n 个关键字的红黑树上，运行花费时间为 O(lgn)。由于这两个操作对树做了修改，结果可能违反红黑树的性质。为了维护这些性质，必须要改变树种某些结点的颜色以及指针结构。

失衡原因

根据 3.1 中的伪代码，应该不难发现造成失衡的原因只有一个就是破坏了性质4，出现了连续的红色结点。也就是当前结点与父节点同时为红色（如果父节点为黑色，不正好符合红黑树所有性质嘛，根本不需要平衡修复）

根据性质4，可以确定的是这时候祖父节点一定为黑色，那么不确定的只有叔结点（如果你还在考虑叔节点会不会存在的问题，建议在去看一下红黑树结构，叔节点要么是内部节点，要么就是值为NIL的叶子节点）

根据性质1，结点只有两种颜色，所以我们可以简单地将情况分为以下两种：

  • 叔节点为红色

但是新的问题又出现了，如果这个祖父结点A的父结点也是那该死的红色呢，那不是又违反性质四了

所以这时候我们的解决方法就是向上递归，俗称甩锅，意思是我们的插入节点就不能仅仅指向插入结点D了，而是要指向它的祖父结点A，然后再根据A的叔结点来判断它属于Case 1还是Case 2进行平衡修复。

思考时间

BST 中我们不能任意移动结点，只能通过旋转的方式改变结点的相对位置。这也就意味着这两个红色结点无论如何都是相连的

(除非你去别的地方投个黑色节点过来，那么这样子不就破坏别人家的黑高了，你说你好好的，去破坏别人家庭干嘛，自己内部的事情当然是自己解决啦)

那么既然z是被我们自己染红的，所以我们只有一条路可以走了，就是把他爸(即f结点)给染黑

可这个时候结点g的左右黑高就不平衡了，破坏了性质5，那我们可以肯定要通过旋转来修复

实践阶段

首先我们染黑结点 f，这一步不管早晚我们都得做，接下去就是解决左子树的黑高多了 1 的这个问题

接下来就不要再有染红祖父结点的冲动啦，这和 case 1 不一样

如果我们染红了祖父节点 g，不仅没有解决树的黑高问题，还使结点 g 可能与其父节点又产生了性质4上的冲突。

这样一来，哪怕我们修复了黑高，还得递归上去，这可不行。

所以我们要对结点 g 来一次 LL 旋转

这时候左子树的黑高问题被转移到了右子树，并且新的根节点 f 与原来一样是黑色，不会向上冲突。

修复

AVL 的平衡修复是 O(lgn)
