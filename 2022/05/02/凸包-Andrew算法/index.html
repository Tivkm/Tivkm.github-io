凸包-Andrew算法

算法发现

在前几个星期的每日一题中，出现了这么一道题，在这里我贴出题目链接，想了解的同学可以去看看 587. 安装栅栏

一开始看到这道题目的时候，我觉得这道题不就是找外围嘛，确实，但是怎么找外围是个很大的难点，我在那里想过来想过去，发现，最靠谱的解法就是—–查看官方题解，这个时候，我就看到了一种解法，叫做凸包解法，经过我百度，凸包解法其实有好几种，包括Jarvis 算法、Graham算法、Andrew算法，在这里，我就说说Andrew算法，其他的算法有空的可以去学学

前置知识

叉乘

在Andrew算法中，有一个比较重要的知识，也是高中学习过的，可能大家都已经忘记了(反正我是忘记了),所以在这里稍微复习一下叉乘的知识点

叉乘的几何定义等于 a x b=|a||b|sinθ，数学定义为 a x b=(x1y2-y1x2)

这时候有小伙伴会奇怪了，不对啊，我明明记得的是a x b = |a| |b| cosθ

确实也有这个公式，不过这个公式是点乘，即a · b = |a| |b| cosθ ,和叉乘不太一样

并且在几何意义也是不一样的

点乘的几何意义是向量a在向量b方向上的投影与向量b的模的乘积(下面这张图角度没标，就是向量a与向量b的夹角)

而叉乘的几何意义则是c是垂直a、b所在平面，且以|b|·sinθ为高、|a|为底的平行四边形的面积，且向量c满足右手定则

在理解了点乘和叉乘的区别后，我们就说说叉乘在这道题里的用处(假如你还理解不了，那就自行百度吧，我相信你的理解能力！！！)

用处

看了上面这张图，你会发现，a x b 和 b x a 不一样，确实，虽然他们在模上是相同的，但是你会发现，他们的方向不一样，在这里我就直接给出结论了，如果你想了解为什么的话，就去查查吧！！

当a x b 大于0时，则向量b在向量a的逆时针方向(通俗来说就是向量b在向量a的上面，当然在左侧的话，向量b就应该在向量a的下面)

那么相反,当a x b 小于0的时候，则向量b在向量a的顺时针方向(像上图，b x a 小于0，则a在b的顺时针方向)

算法讲解

我们围绕着这道题来开始讲解Andrew算法

我在这里讲题目贴出来，感兴趣自己动手先做一遍的同学可以去向上找链接，做不来了或者你不想做就可以直接向下看

题目描述

在一个二维的花园中，有一些用 (x, y) 坐标表示的树。由于安装费用十分昂贵，你的任务是先用最短的绳子围起所有的树。只有当所有的树都被绳子包围时，花园才能围好栅栏。你需要找到正好位于栅栏边界上的树的坐标。

实例一:

思考时间

看完这道题后，我相信你的思考思路肯定和我一样，都是先把点在坐标轴上全画出来，然后把最外围的全部连起来不就好了，确实如此，但是我们怎么去找这个外围呢，Andrew算法就给了我们这样一个思路

首先，先给出这么两个概念，叫上凸壳和下凸壳

上凸壳：上面包围起来的点所连成的线

下凸壳：下面包围起来的点连成的线

这里我就用一个蓝色分割线将该图分成两部分

我们的思路大概是这样的，从左往右求出上凸壳所有的点，在从右往左求出下凸壳所有的点，连起来就是从起点顺时针绕一圈求出所有点

基本流程:

1、对所有点进行排序，先根据x坐标进行升序排序，后面再根据y坐标进行升序排序

根据x坐标进行升序排序的目的是为了我们能够从一个方向画出凸包边缘，而将y升序目的是可以确保一旦两个点相连，能够保证将遍历过的所有的点都能够包裹进去，这里可能讲的有点抽象，稍后我会用图解来告诉你为什么这样子排序

2、使用栈来维护所有凸包上的点，或者说凸包上的边，会更为准确，凸包起点元素会在栈中出现两次（首尾），因此更为准确的描述应该是使用栈维护凸包的所有的边，栈中相邻元素代表凸包上的一条边；

3、分别「从前往后」和「从后往前」处理排序好的所有点，来分别画出凸包的两半部分，根据画的是第一部分还是第二部分，维护栈内元的处理逻辑稍有不同：

  • 画的是上凸壳：

    • 若栈内元素少于 2 个，组成一条线至少需要两个点，说明此时第一条边都还没画出，直接将元素添加到栈中；
    • 若栈内元素不少于 2 个，考虑是否要将栈顶的边删掉（由栈顶前两个元素组成的边）假设栈顶元素为 b，栈顶元素的下一位为 a，即栈顶存在一条 a 到 b 的边，当前处理到的点为 c，此时我们根据 ac 边是否在 ab 边的时针方向来决定是否要将 ab边去掉

    这里就要说到上面为什么我们需要对y进行升序排序，并且这里是非常关键的一步，叉乘的使用也会在这里体现，我们用图一步一步讲解

上凸壳

首先，我们将图中的点全部标出来，接着进行栈的推入，我们直接先推入两个点，表示一个边的确定

这时候当我们推入第三个点的时候，就会发生判断

我们先设两个入栈之间的边为a边，没有入栈的点与前面那个点相连的边为b边，这时候我们发现，由于我们现在需要寻找的是上凸壳（即上面包围的点所形成的线），因此我应该选择b边而不是选择a边，因此这时候就可以用到叉乘性质了，a x b如果大于0，则说明b边在a边的上面，那么我们就取b边，同理，当第四个点进行推入的时候，我们发现该边在b边的上面，因此我们选择的是第四个点

因此这下你就可以明白为什么我们需要以y作升序排列，这样子便可以保证前几个点一定在该范围内

好，那么接下来再继续入栈，这时候情况就和相面不一样了，因为点的横坐标发生了变化

那我们如何判断该点是否进栈，这时候聪明的你一定能够发现，我们只需要判断a边和b边的叉乘就可以知道结果了

你会发现，a x b 是小于0的，不信的话可以自定手动算一下，因此，该点就是得入栈的，因此到现在就变成了这个样子
