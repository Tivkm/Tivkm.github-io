约瑟夫问题

今天的每日一题再次给我重拳一击,说实话，这个问题之前有刷到过，但是忘记了，所以这次碰到就把他记录下来

定义

约瑟夫问题是个著名的问题：N个人围成一圈，第一个人从1开始报数，报M的将被杀掉，下一个人接着从1开始报。如此反复，最后剩下一个，求最后的胜利者。
例如只有三个人，把他们叫做A、B、C，他们围成一圈，从A开始报数，假设报2的人被杀掉。

  • 首先A开始报数，他报1。侥幸逃过一劫。
  • 然后轮到B报数，他报2。非常惨，他被杀了
  • C接着从1开始报数
  • 接着轮到A报数，他报2。也被杀死了。
  • 最终胜利者是C

解决方案

普通方法

刚看到这道题的时候，你会觉得这个和循环链表非常类似，首尾相接，然后通过遍历进行删除节点，最后当链表中只有一个节点的时候，他就是胜利者。

缺点

要模拟整个游戏过程，时间复杂度高达O(nm)，当n，m非常大(例如上百万，上千万)的时候，几乎是没有办法在短时间内出结果的。

公式法

约瑟夫环是一个经典的数学问题，我们不难发现这样的依次报数，似乎有规律可循。为了方便导出递推式，我们重新定义一下题目。
问题： N个人编号为1，2，……，N，依次报数，每报到M时，杀掉那个人，求最后胜利者的编号。

这边我们先把结论抛出了。之后带领大家一步一步的理解这个公式是什么来的。

结论

f(N,M) = (f(N-1,M)+M) % N

  • f(N,M) 表示，N个人报数，每报道M时杀掉那个人,最终胜利者的编号
  • f(N,M) 表示，N-1个人报数，每报道M时杀掉那个人,最终胜利者的编号

递推过程

我们首先来模拟这个场景

下面我们不用字母表示每个人，而用数字

​ 1、2、3、4、5、6、7、8、9 M = 3

表示有8个人,假设每报到的人被杀掉(这会不会被和谐掉,有点过于残忍)

  • 刚开始，头一个编号是1，他开始报数，第一轮被杀掉的是编号3的人
  • 编号4的人开始重新报数，这时候我们可以认为编号4这个人是队伍的头。第二轮被杀掉的是编号6的人。
  • 。。。。
  • 第7轮时，编号7的人开始重新报数，我们以7为头，这轮被杀掉的是2
  • 第八轮，他从1开始报数，不幸的是他在这轮被杀掉了
  • 最后的胜利者是编号为1的人

思考过程

我们认真看一下每一轮生存者的位置变化情况

细心的你肯定发现了，每次一个人被杀掉的时候，存活者全部向前移动了m个位置

例如在第一轮的时候，当编号3的人被杀死后，所有的数都向前移动了3格,最前面的数就被排到了后面(当然，移动的时候被杀的人的位置也是要考虑进去的)

这时候你就又有疑问了，当从头排到尾的时候，是排在所有队列的最后，还是排在现在有生存的人的后面呢？

你去看一下上面那张图就知道了，应该排在当前还剩的生存的人后面

因此公式就出来了

生存者在被杀之人杀死之后的位置 = (生存者在被杀之人杀死之前的位置 - 报数 + 当前有多少生存人的个数) % 当前有多少生存人的个数

好，现在又有个问题出来了，我一开始又不知道最后的胜利者是谁，难道要从头到尾遍历一遍嘛？

不不不，虽然我们一开始不知道胜利这是谁，但是我们最终不是会把胜利者是谁给求出来嘛，因此，我们反向推倒不就可以把答案算出来了嘛，当然正向推是向前移动m个位置，那反向推不就向后移动m个位置嘛，并且我们只需要关心胜利者的位置，其他的人反正最终都会被杀掉

在这里我把逆推的过程在重复一遍，相信你能一下子就看懂

  • f(1,3) = 0 只有1个人了，那个人就是获胜者，他的下标位置是0
  • f(2,3) = (f(1,3)+3)%2=3%2=1 在有2个人的时候，胜利者的下标位置为1
  • f(3,3) = (f(2,3)+3)%3=4%3=1 在有3个人的时候，胜利者的下标位置为1
  • …….
  • f(9,3) = 0
